# Content Gift — 要件定義書

## 1. プロダクト概要

| 項目 | 内容 |
|------|------|
| アプリ名 | Content Gift |
| 目的 | リスト獲得・セミナー参加率を最大化する特典コンテンツをAIで設計・生成 |
| 対象ユーザー | プロモーター / コンサルタント / コーチ / 講座運営者 |
| 利用形態 | ブラウザアプリ（PC・タブレット）。ローカルまたはサーバーで動作 |

## 2. 解決する課題

| 課題 | 本アプリでの解決策 |
|------|-------------------|
| どんな特典を作ればいいかわからない | AIが商品情報から6つのアイデアをスコア付きで提案 |
| 特典が「ありきたり」で刺さらない | 競合分析＋6指標評価で差別化された特典を設計 |
| 特典を作る時間がない | AIがコンテンツ本文を自動生成（チェックリスト・動画台本・ガイドブック等） |
| 特典の品質が判断できない | 定量・定性の両面からAIが100点満点で評価＋改善提案 |
| 特典から有料商品につながらない | 「架け橋戦略」を特典設計に組み込み、自然な導線を設計 |
| データが消えるのが怖い | バックアップ/復元機能でJSON書き出し・読み込み |

## 3. 機能一覧

### 3.1 プロジェクト管理
- プロジェクト作成
- プロジェクト一覧表示（アイデア数・コンテンツ数・用途）
- プロジェクト削除
- データバックアップ（全プロジェクトをJSONエクスポート）
- データ復元（JSONインポート。既存は上書き、新規は追加）
- アプリ内使い方マニュアル（ヘッダーの「📖 使い方」からアクセス）

### 3.2 商品設定（Config）
- 商品・サービス情報入力（13項目）
  - 商品名、商品概要、ターゲット、悩み、理想、価格帯、競合の特典、希望アクション、権威性
  - ターゲットの知識レベル（初心者 / 中級者 / 上級者）
  - ターゲットの緊急度（情報収集中 / 検討中 / 今すぐ欲しい）
  - 達成したい目標（コンバージョンゴール）
- 特典の用途選択（リスト獲得用 / セミナー特典 / 両方）
  - 用途ごとにAIの設計原則が自動切替（ファネルステージ別プロンプト最適化）
- 希望する特典形式選択（AIにおまかせ / 6種類から選択）
- テキスト一括入力（LPテキストの貼り付け）
- 他ツールで設計済みのコンセプト・ファネル・セミナー内容の入力（AI連携用）

### 3.3 アイデア生成
- 商品情報からAIが6つの特典アイデアを自動提案
- 各アイデアに6指標のスコア（100点満点）
  - リスト獲得力 / 感動指数 / 架け橋度 / 即実践度 / セミナー参加率UP / 制作しやすさ
- 各アイデアに含まれる情報
  - タイトル、概要、フック（キャッチコピー）、なぜ効くか、架け橋戦略、構成案
- ワンタップコピー（フック）
- 最高スコアに「👑 おすすめ」表示
- タイトル案5パターン生成（数字×結果型 / 共感型 / 好奇心型 / 即効ツール型 / Before/After型）
  - ワンクリックでタイトル差し替え
- 再生成可能

### 3.4 コンテンツ生成
- 選択したアイデアからAIが特典コンテンツの本文を自動生成
- 対応する6種類の形式
  - ✅ チェックリスト（診断系、やることリスト系）
  - 📖 ガイドブック/PDF資料（ステップバイステップ）
  - 🎬 動画台本（話し言葉、演出指示付き）
  - 📝 ワークシート（書き込み式、質問・記入例付き）
  - 📋 テンプレート集（穴埋め式のひな形）
  - 💎 スワイプファイル（成功事例集）
- 生成コンテンツの構成
  - タイトル＆サブタイトル
  - はじめに（読者への語りかけ）
  - 本編セクション（8〜12セクション）
  - おわりに（感謝と次のステップ）
  - CTA（有料商品への自然な導線）
- コピー機能
  - 各セクション個別コピー
  - 全文一括コピー
  - Markdown形式コピー（構造化テキスト）
  - デザインツール用コピー（スライド構成）
- PDF出力対応（動画台本のみ。A4・表紙付き・形式別レイアウト）
  - 資料系（チェックリスト・テンプレート等）はデザインツールでの制作を推奨
- 再生成可能

### 3.5 品質チェック
- AIが定量・定性の両面から100点満点で評価
- 定量評価（5項目）
  - 文字数、セクション数、アクションアイテム数、具体例数、データポイント数
- 定性評価（6項目、各10点満点）
  - 明瞭さ / 即実践性 / 独自性 / 感情インパクト / 知覚価値 / 架け橋効果
- 判定: Excellent / Good / Needs Improvement
- 強み（3〜5つ）
- 改善ポイント（優先度付き）
- 書き換え提案（Before → After形式）
  - 各提案に該当箇所（📍 セクション名）を表示
  - ワンクリック自動適用ボタン（個別 / 一括）
- 複数コンテンツの一括品質チェック（バッチ処理）
  - 進捗バー表示
  - コンテンツ一覧スコア表示
- 再分析可能

## 4. 技術要件

### 4.1 技術スタック
| 項目 | 技術 |
|------|------|
| フレームワーク | Next.js 15 (App Router) |
| 言語 | TypeScript |
| スタイリング | Tailwind CSS v4 |
| AI | Anthropic Claude API (claude-sonnet-4-20250514) |
| データ保存 | localStorage（ブラウザ内） |

### 4.2 外部依存
| サービス | 用途 | 必須 |
|---------|------|------|
| Anthropic API | アイデア生成、コンテンツ生成、品質チェック | 必須 |
| Google Fonts | Noto Sans JP フォント | 推奨 |

### 4.3 環境要件
| 項目 | 要件 |
|------|------|
| Node.js | v18 以上 |
| ブラウザ | Chrome / Safari / Firefox / Edge（最新版） |
| API キー | Anthropic API キー（`ANTHROPIC_API_KEY`） |

## 5. 品質基準（特典コンテンツの）

### 定量基準
| 項目 | 目標値 |
|------|--------|
| 文字数 | 8,000〜15,000文字（動画台本は10,000〜18,000文字） |
| セクション数 | 8〜12 |
| アクションアイテム | 20個以上 |
| 具体例・事例 | 10個以上 |
| データ・統計 | 5個以上 |
| Before/After事例 | 3個以上 |

### 定性基準
| 項目 | 基準 |
|------|------|
| 知覚価値 | 有料5,000〜10,000円で売れるレベル |
| 即実践性 | 読んだ直後に行動できる具体性 |
| 架け橋効果 | 「もっと学びたい」→有料商品への自然な導線 |
| 独自性 | ググれば出てくるレベルを超えた専門性 |

## 6. セキュリティ

| 項目 | 対応 |
|------|------|
| データ保存場所 | ブラウザのlocalStorage（サーバーには送信しない） |
| API通信 | Anthropic APIへのサーバーサイド通信のみ（APIキーはサーバー側で保持） |
| バックアップ | JSONファイルとしてローカルに保存。暗号化なし |

## 7. 画面構成

```
トップページ（プロジェクト一覧）
├── 🎁 特典コンテンツを作る（新規作成）
├── 📖 使い方マニュアル → /manual ページ
├── バックアップ / 復元
└── プロジェクトカード → 詳細ページへ

📖 使い方マニュアル（/manual）
├── 全体の流れ
├── 商品情報を設定する
├── アイデアを生成する
├── コンテンツを生成する
├── 品質チェック
├── PDF保存
├── バックアップと復元
└── 効果的な特典の作り方

プロジェクト詳細
├── ⚙️ 設定タブ（商品情報 + ターゲット深掘り + コンバージョン設定）
├── 💡 アイデアタブ
│   ├── 6つのアイデア（スコア付き）
│   ├── タイトル案5パターン生成
│   └── アイデアからコンテンツ生成
├── 📄 コンテンツタブ
│   ├── 生成コンテンツ表示
│   ├── コピー機能（全文 / Markdown / デザインツール用）
│   └── PDF保存（動画台本）
└── ✅ 品質チェックタブ
    ├── コンテンツ一覧スコア表示
    ├── 一括品質チェック
    ├── 定量・定性評価
    ├── 改善ポイント
    └── 書き換え提案（自動適用対応）
```
